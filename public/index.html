Many to Many PostgreSQL

Node backend API with many to many association - postgreSQL and sequelize ORM. API is posted on AWS with EC2 instances and RDS database

The API consists of two related models. Movie model and Actor. Api can be tested using the Postman app.

A route can be used to get a list of all teams:

GET http://18.185.73.229/api/v1/teams

A route can be used to get a single team:

GET http://18.185.73.229/api/v1/teams/:slug

A route can be used to get a list of all coaches:

GET http://18.185.73.229/api/v1/coaches

A route can be used to get a single coach:

GET http://18.185.73.229/api/v1/coaches/:slug
First you need to create all the football teams you want to belong to your API, and then join the coaches. The API is constructed in such a way that one team cannot have two coaches, as well as that one coach cannot lead two clubs.

When creating a team, it is necessary to input the name of the club, as well as the city and country in which it is located

  • teamName: ""
  • city: ""
  • country: ""

A route can be used to post team:

POST http://18.185.73.229/api/v1/teams

When creating a coach, it is necessary to input the slug of the club, as well as first name and last name of coach

  • teamSlug: ""
  • firstName: ""
  • lastName: ""

A route can be used to post coach:

POST http://18.185.73.229/api/v1/coaches
If you want to change the data related to teams or coaches, you can do so as follows:

When updating a team, it is necessary to input the new name of the club

  • teamName: ""

A route can be used to post team:

PUT http://18.185.73.229/api/v1/teams/:slug

When updating a team coach, it is necessary to input the team slug and old team coach name.

  • teamSlug: ""
  • oldCoachSlug: ""

A route can be used to post coach:

PUT http://18.185.73.229/api/v1/coaches/:slug

If the team is currently without a coach, assign it when creating a new coach. You have to keep in mind that a coach who has already been assigned to a team cannot be the coach of two clubs

If you want to delete a football team or coach, you can do so in the following way:

A route can be used to delete team:

DELETE http://18.185.73.229/api/v1/teams/:slug

A route can be used to delete coach:

DELETE http://18.185.73.229/api/v1/coaches/:slug
