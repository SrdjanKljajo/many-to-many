Many to Many PostgreSQL

Node backend API with many to many association - postgreSQL and sequelize ORM. API is posted on AWS with EC2 instances and RDS database

The API consists of two related models. Movie model and Actor. Api can be tested using the Postman app.

A route can be used to get a list of all movies:

GET http://3.125.160.201/api/v1/movies

A route can be used to get a single movie:

GET http://3.125.160.201/api/v1/movies/:slug

A route can be used to get a list of all actors:

GET http://3.125.160.201/api/v1/actors

A route can be used to get a single actor:

GET http://3.125.160.201/api/v1/actors/:slug
First you need to create actors and movies. The API is constructed in such a way that only actors and movies who is already in database may be related.

When creating a actor, it is necessary to input the name of the actor.

POST http://3.125.160.201/api/v1/actors

When creating a movie, it is necessary to input the title of the movie, as well as director name, release year, and duration of movie in number of minutes

A route can be used to create movie:

If you want to change the data related to actors or movies, you can do so as follows:

When updating a actor, it is necessary to input the new name of the actor

A route can be used to update actor:

PUT http://3.125.160.201/api/v1/actors/:slug

When updating a movie, it is necessary to input the new movie data

  • title: ""
  • director: ""
  • release: ""
  • duration: ""

A route can be used to update movie:

PUT http://3.125.160.201/api/v1/movies/:slug
If you want to delete a actor or movie, you can do so in the following way:

A route can be used to delete actor:

DELETE http://3.125.160.201/api/v1/actors/:slug

A route can be used to delete movie:

DELETE http://3.125.160.201/api/v1/movies/:slug
